Git - Aufstieg der Versionsverwaltung

"Wie bei vielen großen Dinge im Leben, begann Git mit ein wenig kreativen Zerstörung und feurige Kontroverse." - Git Entwicklung
Die Entwicklung des Linux Kernels als Open Source Softwareprojekt umfasst einen immer weiteren Bereich, sowohl an verteilter Softwareentwicklung und globaler Distribution. Die Verbreitung der Änderungen an diesem Projekt erfolgten in der anfänglichen Entwicklungszeit als Patches und Archivdateien. (1991 - 2002)

2002 begann innerhalb des Entwicklugnsprojekts des Linux Kernels die Benutzung einer verteilten Versionsverwaltung namens BitKeeper.

Nach einiger Zeit der Zusammenarbeit zwischen der Linux entwickelnden Community und der kommerziellen Entwicklerfirma von BitKeeper wurde diese durch die Entzweiung der beiden Interessengruppen beendet. Diese Ereignisse gaben der Entwicklercommunity, und besonders Linus Torvalds , Schöpfer von Linux, Anlass, um eine eigene Software zur Versionsverwaltung zu entwicklen. Diese profitierte von den Erfahrungen, die durch die Benutzung von BitKeeper errungen wurden. Dazu wurden folgende Eigenschaften als Ziele festgelegt:
  • Geschwindigkeit
  • Einfaches Design
  • Optimale Unterstützung für nicht-lineare Entwicklung (parallel branches)
  • Vollständige Verteilung
  • Effiziente Nutzung für globale Projekte wie den Linux Kernel (bezüglich Geschwindigkeit und Datengröße)
